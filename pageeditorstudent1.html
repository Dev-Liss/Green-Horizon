Splash Page

Technical Description

HTML Structure and Functionality

<meta> Tags:
- Ensures character encoding (UTF-8) and responsive design (viewport).
- Implements automatic redirection (<meta http-equiv="refresh" content="4; url=home.html">), ensuring users transition to home.html after 4 seconds.

External Libraries:
- Uses Font Awesome for icons.
- Google Fonts (Poppins) for modern typography.
- Animate.css for animations.

Key Containers:
- .zoom-container & .zoom-image: Displays and animates a background image using CSS.
- .splash-container: Main container positioned fixed, ensuring full-screen coverage.
- .splash-content: The content box with rgba background for transparency and rounded corners.

Team Section:
- .members-list: Displays team names as styled pills.
- .member: Styled name badges with background color.

Loading Animation:
- .loader: The loading bar with a background color.
- .loader-progress: Animates from 0% to 100% width in 4s.

CSS Styling and Effects

Global Styles:
- Resets margins, paddings (* selector).
- Implements CSS variables (:root) for color consistency.

Typography & Colors:
- Uses Poppins font with varying weights.
- Ensures readability with contrasting colors.

Background and Animation:
- .zoom-image scales between 1x and 1.2x using @keyframes slow-zoom (10s duration).
- .loader-progress animates width (4s).

Responsiveness:
- Uses media queries to adjust padding, font sizes, and container width for mobile screens.

JavaScript
- Applies a fade-out effect before redirection.

Accessibility

Accessibility Features in the Splash Screen Page

1. Semantic HTML:
- Used proper tags like <h1> for titles and <p> for text to improve screen reader compatibility.

2. Text Readability & Contrast:
- Ensured high contrast between text and background for visibility.
- Used scalable font sizes (rem) so users can adjust text size as needed.

3. Alternative Text for Images:
- Included an alt attribute for images (<img src="..." alt="Zooming image">) to describe content for visually impaired users.

4. Keyboard & Screen Reader Support:
- Ensured interactive elements can be navigated using a keyboard.
- Avoided empty <div> elements to prevent confusion for screen readers.

5. Motion Sensitivity Considerations:
- Used smooth, non-intrusive animations (@keyframes slow-zoom) to avoid discomfort for users with motion sensitivities.

Volunteer Page

Technical Description

1. HTML Structure & Key Tags
-The HTML file follows a structured format with semantic elements for readability and accessibility.
1.1 Key Sections and Their Purpose:
- Head Section (<head>)
  • Contains metadata for character encoding (<meta charset="UTF-8">) and responsive design (<meta name="viewport" content="width=device-width, initial-scale=1.0">).
  • Links to external stylesheets and FontAwesome for icons.
- Navigation Bar (<nav>)
  • Uses <ul> and <li> elements for menu items.
  • A hamburger menu (<div class="menu-toggle">&#9776;</div>) is included for mobile navigation.
  • The logo inside <div class="logo"> is clickable, leading to home.html.
- Sorting Box (<div class="sort-box">)
  • Allows users to sort programs using a dropdown (<select>).
- Volunteer Program Cards (<div class="cards-container">)
  • Uses <div class="card"> for individual volunteer programs, with an image, heading, description, and "Learn More" button.
- User Reviews (<section class="reviews">)
  • Displays review cards (<div class="review-card">) with ratings and testimonials.
- Feedback Form (<section class="feedback-section">)
  • Allows users to rate their experience using emoji buttons.
  • Allows users to submit feedback through a textarea.
- Footer Section (<footer>)
  • Includes categorized links and social media icons styled using FontAwesome.
JavaScript Inclusion (<script src="./js/script.js"></script>)
  • Suggests additional interactive behavior (e.g., sorting functionality or "Go to Top" button).
2. CSS Styling & Techniques
General Styles

1. Font & Colors
- body { font-family: Arial, sans-serif; color: #ffffff; line-height: 1.6; }
- Ensures a clean, readable text style.

2. Background Image for the Page
.volunteer-page { background-image: url("../images/volunteer_page/beach.jpg"); background-size: cover; background-position: center; background-attachment: fixed; background-repeat: no-repeat; }
- Provides a full-page background image with a fixed position, ensuring a consistent visual effect.

3. Content Container (.content-container)
background-color: rgba(0, 0, 0, 0.7); border-radius: 10px;
- A semi-transparent black background improves text readability.

4. Cards & Reviews (Volunteer Programs & Testimonials)
- Flexbox Grid Layout for program cards
.cards-container { display: flex; flex-wrap: wrap; justify-content: center; gap: 10px; }

5. Card Hover Effect to improve interactivity:
.card:hover { transform: translateY(-5px); box-shadow: 0 8px 16px rgba(0, 0, 0, 0.2); }
- Gives a floating effect when hovering over a card.

6. Sorting Box
- Dropdown Styling with a Custom Icon:
.sort-box select { background-image: url("data:image/svg+xml;utf8,<svg fill='%234CAF50' ...>"); background-repeat: no-repeat; background-position: right 8px center; }
- Custom dropdown design improves usability.

7. Review Cards
- Styled similarly to program cards but smaller:
.review-card { background-color: white; border-radius: 10px; padding: 20px; box-shadow: 0 6px 12px rgba(0, 0, 0, 0.3); }

8. Reviewer Details Formatting:
.review-card .reviewer { font-size: 0.8rem; color: #999; }

9. Feedback Section
- Uses flexbox for form layout:
.feedback-form { display: flex; flex-direction: column; gap: 20px; }

10. Emoji Rating Buttons with Hover Effects:
.emoji-btn:hover { transform: scale(1.1); border-color: #4CAF50; background-color: #f0f9f0; }

11. "Go to Top" Button
- Positioned at the bottom right of the screen:
#goToTopBtn { position: fixed; bottom: 20px; right: 30px; z-index: 99; border-radius: 50%; }

12. Changes color on hover:
#goToTopBtn:hover { background-color: #45a049; }
- Easy to understand selcted or not

3. Responsive Design
- Uses media queries to adjust layouts for different screen sizes:
@media (max-width: 1024px) { .card, .review-card { width: calc(50% - 20px); } }
@media (max-width: 768px) { .card, .review-card { width: 100%; } .sort-box { justify-content: center; } }
- Cards stack vertically on smaller screens.
- Feedback buttons adjust for better touch usability.

Accessibility

Accessibility Features in the Volunteer Page
The volunteer page was designed with accessibility in mind to ensure an inclusive user experience.

1. Semantic HTML
- The page uses <header>, <nav>, <main>, <section>, and <footer> elements to provide a clear document structure for screen readers.

2. Keyboard Navigation
- Interactive elements such as buttons, links, and dropdowns are accessible via the keyboard using the tab key.

3. Alt Text for Images
- All images have descriptive alt attributes to assist visually impaired users in understanding content.

4. Sufficient Color Contrast
- Text and background colors maintain high contrast, ensuring readability for users with visual impairments.

5. Responsive Design
- The layout adapts to different screen sizes, improving usability for users on mobile devices.

6. Form Accessibility
- Labels are associated with form inputs, and placeholder text provides clear guidance for user interactions.

7. ARIA Attributes
- Accessible Rich Internet Applications (ARIA) attributes are used for dynamic elements to improve screen reader support.

Challenges and Lessons Learned

Challenges Faced and Solutions

1. Responsive Design Issues: The navigation menu and content layout did not adjust well on smaller screens.
Solution: Used flexbox and media queries to improve adaptability, ensuring sections reflow properly.

2. Accessibility Improvements: Ensuring proper screen reader support and keyboard navigation was difficult.
Solution: Implemented semantic HTML, added descriptive alt attributes for images, and ensured keyboard focus on interactive elements.

3. Performance Optimization: The splash screen caused delays due to heavy animations.
Solution: Optimized animations and minimized resource usage for a smoother experience.

Lessons Learned: Planning for responsiveness and accessibility from the beginning results in a more user-friendly and maintainable design.

Compliance

Compliance with Janet Web Page Publication Regulations:
Our web pages have been developed with adherence to key Janet regulations governing web publication, ensuring security, accessibility, and ethical content distribution.

1. Accessibility Compliance: We implemented semantic HTML, proper heading structures, alt text for images, and keyboard navigability to align with accessibility standards, making the site usable for individuals with disabilities.

2. Data Privacy & Security: No sensitive user data is collected or stored within these pages. External links and scripts are used responsibly, and security measures such as HTTPS are prioritized to prevent vulnerabilities.

3. Ethical & Legal Content: The content adheres to ethical guidelines by providing accurate, well-researched information without misleading claims. All media assets, including images and icons, either belong to the project or are sourced from free-to-use platforms with proper attribution.

4. Performance & Network Efficiency: The pages are optimized to ensure fast loading times, reducing unnecessary resource consumption and complying with Janet’s network efficiency policies.

Understanding & Application: By following these regulations, we ensure that our web pages maintain ethical, legal, and technical standards while prioritizing user experience and accessibility.
