Feedback Form

Technical Description

1. HTML Structure and Functionality

  • Meta Tags: UTF-8 encoding, responsive viewport.
  • External Libraries: Google Fonts (Poppins, Belanosima).
  • Form Components:
    • Input fields for Name, Email, Phone Number, Address.
    • Radio buttons for experience feedback.
    • Dropdown for update preferences.
    • Text area for additional comments.
    • Emoji-based rating system.

2. CSS Styling and Effects

  • Global Styles: Resets default margins/paddings.
  • Typography & Colors: Uses Poppins font, high contrast.
  • Form Styling: Structured layout, styled buttons.
  • Responsiveness: Flexible container adapts to screens.

3. JavaScript Features

  • Validates Name (only letters), Email (proper format), and Phone Number (10 digits).
  • Prevents submission if validation fails.
  • Displays alerts for invalid input.

4. Accessibility Features

  • Semantic HTML: Uses <label> for better screen reader support.
  • Text Readability: Ensures contrast for better visibility.
  • Keyboard & Screen Reader Support: Fully navigable via keyboard.
  • Validation Feedback: Alerts users to errors.

Challenges and Lessons Learned

Challenges Faced and Solutions

1. Designing an Interactive Feedback Form: Ensuring that the feedback form was user-friendly while capturing all the necessary data.
Solution: Created a responsive form layout with a clear structure for inputs and radio buttons. Added real-time form validation using JavaScript to improve user experience.

2. Ensuring Accessibility and Usability: The form needed to be accessible for users with different needs and screen sizes.
Solution: Incorporated semantic HTML for accessibility, and used CSS media queries to make the form responsive on various devices.

3. Validating User Input: Validating user input on fields like name, email, and phone number for accurate data submission.
Solution: Implemented JavaScript validation to ensure correct user inputs before form submission.

Lessons Learned: The importance of form validation and the need for a responsive design became evident. Ensuring accessibility and usability for a wider audience was a key takeaway.

Compliance

  • Accessibility Compliance: Ensured web pages are accessible by using semantic HTML, providing alternative text for images, and enabling keyboard navigation for users with disabilities.

  • Data Protection and Privacy: Adhered to data protection guidelines by minimizing data collection, not requesting sensitive information, and ensuring the privacy of user data.

  • Content Accuracy: Ensured all content is clear, accurate, and free from misleading or inappropriate information, adhering to ethical web standards.

  • Compliance with Web Standards: Followed web standards for structure, accessibility, and ethical guidelines, ensuring the pages meet the required regulations.
